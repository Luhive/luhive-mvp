## Image Crop Upload Refactor

Fixed profile photo not updating in the UI until refresh, added square crop to profile uploads, and extracted shared crop utilities used across upload components.

### Bug fix — instant profile photo update

- `profile-picture-upload.tsx` — `AvatarImage` was bound to `currentLogoUrl` (prop) instead of `previewImage` (local state), so the avatar never reflected uploads until a full page refresh; now uses `previewImage`
- Removed `useRevalidator().revalidate()` — local preview state handles the instant update

### Shared crop utilities

- **Added** `shared/lib/utils/image-crop.ts` — `createImage`, `getCroppedImg`, and `CroppedAreaPixels` type extracted from duplicated canvas logic in cover/event upload components

### Shared crop dialog

- **Added** `shared/components/image-crop-dialog.tsx` — reusable crop dialog (`react-easy-crop`, lazy-loaded for SSR) with configurable `aspect`, `title`, `hint`, `objectFit`, zoom slider, upload progress, and save/cancel/remove actions; no storage or domain logic inside

### Profile picture upload

- `profile-picture-upload.tsx` — file selection opens `ImageCropDialog` (1:1 square, 400×400 hint); cropped blob uploaded via existing `uploadCommunityLogo`; syncs `previewImage` and `onLogoUpdate` on success
- Syncs `previewImage` from `currentLogoUrl` prop when it changes

### Cover & event upload refactors

- `cover-picture-upload.tsx` — replaced inline crop dialog and duplicate helpers with `ImageCropDialog` + `getCroppedImg` (4:1 aspect, `horizontal-cover`); upload/remove logic unchanged
- `event-cover-upload.tsx` — same refactor (1:1 aspect, `contain`); creation vs edit DB update behaviour unchanged
