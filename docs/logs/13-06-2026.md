## Azerbaijani-aware slugs

ASCII-only slugify was stripping AZ letters (ə, ı, ş, …) and producing empty/`event-{uuid}` slugs.

- **Added** `app/shared/lib/utils/slug.ts` — `toLocaleLowerCase('az')`, transliterate to ASCII, `slugifyAscii`
- `event-slug.ts` delegates to shared util; reserved-word suffix unchanged
- `20260613` migration updated with `slugify_event_title()` SQL function (AZ case-fold + `translate`)
- **Migration** `20260614_repair_event_slugs_az.sql` — re-slug all existing events with same logic (for DBs that already ran the first migration)

## Event slug URL showing community page

`/c/luhive2/why-audience-matter` rendered the community dashboard because `isChildRoute` only checked `/announcements` and `/events`.

- `Routes.community.isCommunityChildPath(slug, pathname)` — true for any nested path under `/c/:slug/`
