# 06-07-2026

## Sidebar — quick access to your profile

From the dashboard sidebar, you can now open your personal profile directly from the user menu.

- Click your name/avatar at the bottom of the sidebar to open the menu
- Tap the user info at the top of that menu (avatar, name, email) to go to your profile page
- Log out option is unchanged

## Community profile — Discord link

Communities can now add a Discord invite/link alongside their other social accounts.

- **Edit profile (dashboard):** new Discord field under Social Media accounts, with the Discord icon
- **Public community page:** Discord link appears with the other social buttons when one is saved
- Discord icon is shown in grayscale so it matches the look of the other social icons

## Event registration — sign in with Google

Guests can now register for an event using Google, the same way they can join a community — without going through email OTP first.

- **Register overlay:** new “Continue with Google” option on the first step, alongside email registration
- **Simple events (no extra questions):** after Google sign-in, registration completes automatically and the user lands back on the event page as registered
- **Events with custom questions:** Google sign-in first, then the user is sent back to the register overlay to answer the required questions before registration is finalised
- **Join community:** the existing checkbox still applies — if unchecked before Google, the user registers for the event but does not join the community
- **Approval-required events:** registration goes to pending approval; the user gets the same pending flow as email registration
- **Already registered:** signing in again does not create a duplicate registration
- **Invites:** invitees who arrive through the invite flow still complete registration via the invite path (custom questions only) — they do not see the Google option, since their email is already verified through the invite

## Event registration — simpler email flow

Email registration for events is now fewer steps: one form, then email verification — instead of separate screens for email, name, and questions.

- **Register overlay:** choose Google, or expand **or with email** to open the full form on the same page
- **Email path:** email, full name, phone, and any custom questions are collected together, then the user enters the one-time code to finish
- **Google path:** unchanged — sign in with Google, then answer custom questions if the event has them
- **Already have an account:** same single form; if the email is known, verification still happens via OTP without creating a duplicate registration

## Event registration — expandable email form

The email registration form no longer shows every field up front — it opens when the guest chooses email.

- **First view:** Register with Google, plus a short **or with email** link
- **After tapping email:** the form expands smoothly into the full fields and Register button
- **Spacing:** the divider and fields sit directly under Google without an awkward gap when expanded

## Event registration — Google profile picture

When someone registers or signs in with Google, their Google profile photo can become their Luhive avatar.

- **New Google users:** profile picture is saved automatically when Google provides one
- **Returning users:** Google photo is only added if they do not already have a Luhive avatar — existing custom pictures are not replaced
- **Events with custom questions:** after Google sign-in, the register screen can show the user’s photo next to their name while they complete the form

## Event registration — add to calendar

Registered attendees can now add an event to their calendar directly from the event page, without relying only on the confirmation email.

- **Registration card:** new **Add my calendar** button for approved registrations, alongside Invite a Friend, Your QR, and Join Event Chat
- **Desktop:** opens a modal with Google Calendar, Outlook, and iCal options
- **Mobile:** same choices in a bottom sheet
- **Helper copy:** reminds guests the invite was already emailed and to check their **spam folder**
- **Google Calendar / Outlook:** open the provider with event details pre-filled
- **ICal:** downloads an `.ics` file for Apple Calendar and other apps
- **Outlook timing:** start and end times match what is shown on the event page
- **Modal layout:** cleaner list-style provider options with icons and dotted-underline labels instead of large coloured buttons
